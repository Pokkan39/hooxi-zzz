# Hooxi 绝区零剧情档案站

## 当前版本

这是一个不依赖构建工具的静态网站，包含绝区零主题路线图、章节展开/折叠、动态背景、站内音乐播放器和“小白可用”的可视化编辑器。内容按照绝区零官方 Wiki 首页可见的常用栏目整理为：主线剧情、角色剧情分支、角色专属 EP/PV、主题活动。站内条目是 Hooxi 档案的可编辑示例，不代表 Hooxi 的真实投稿。

## 小白编辑流程

1. 打开网站，点击右上角黄色 `✦`，进入“站点编辑器”。
2. **外观**：修改主标题、首页介绍、背景图片 URL、透明度和动态效果开关；点击“应用外观”即可立即生效。
3. **内容**：修改已有剧情卡片的分类和标题，或点击“新增剧情卡片”。保存后路线图立即刷新。视频地址可在 `app.js` 的数据中继续补充，或后续扩展编辑字段。
4. **歌单**：填写歌曲名称和你有权使用的 MP3/OGG 地址；保存后底部播放器可以播放、切换、调节音量。
5. **导出配置**：把当前所有内容保存为 JSON 备份；换电脑时用“导入配置”恢复。需要回到初始状态时点击“恢复默认”。

配置会保存在当前浏览器的 `localStorage` 中，不需要数据库。若清理浏览器数据，先导出配置备份。

## 音乐与自动播放说明

播放器会在页面加载时尝试自动播放，但现代浏览器通常会阻止带声音的自动播放，这是浏览器安全策略，无法通过网页可靠绕过。出现提示时点击底部播放按钮即可启用。请只使用自己创作、获得授权或明确允许使用的音频地址。

## 词条封面、角色立绘与 Wiki 字段

四个档案页的封面现在支持点击跳转视频：词条同时填写“封面图片”和“视频链接”后，访客点击封面会在新标签页打开视频。封面图片建议放到 `assets/covers/`，角色头像/立绘建议放到 `assets/portraits/`，编辑器中填写相对路径。

编辑器按官方 Wiki 常见的信息组织方式分为：基本信息（分类、类型、标题、简介、版本、日期）、角色与世界观（阵营、相关角色、地点/空洞区域）、媒体与来源（视频、封面、角色立绘、附加图片、官方资料链接）。这些字段是结构化预留，不会自动复制 Wiki 原文；可参考：

- <https://wiki.biligame.com/zzz/游戏历史>
- <https://zenless-zone-zero.fandom.com/zh/wiki/绝區零_Wiki>

选择本地图片按钮只提供当前浏览器预览，静态 GitHub Pages 不会自动上传文件。要让所有访客看到，仍需把图片复制到对应资源目录，并在字段中填写路径后导出 `data.js` 发布。图片只保存路径，使用懒加载，适合后续增加数百张图片；封面采用完整展示模式，不会裁切横图或竖图主体。


网站现在提供四个独立档案页：主线时间轴、支线/角色故事、代理人 EP/PV、主题活动。每个页面的卡片都可以用鼠标拖动重新排序，排序保存在当前浏览器；`data.js` 是默认数据源。

四个时间轴页面右上角的 `✦` 都有独立页面编辑器，可以修改标签、标题、简介和视频地址，也可以新增、删除条目。B 站视频地址支持直接粘贴完整链接或 BV 号；页面会尝试读取公开视频封面，若接口受限则显示提示，但仍可点击链接打开 B 站。

### 让所有访客看到编辑结果

1. 在任意一个时间轴页面打开右上角 `✦` 编辑器。
2. 编辑内容后点击“导出发布数据 data.js”。该文件包含四个页面的完整数据。
3. 用下载的 `data.js` 覆盖仓库根目录的同名文件。
4. 提交并推送到 GitHub，等待 GitHub Pages 部署完成，所有访客就能看到新内容。
5. 以后可在编辑器中选择“导入 data.js / JSON”恢复数据；导入只更新当前浏览器，仍需再次导出并发布才能同步给访客。

编辑内容保存在当前浏览器的 `localStorage`，不会自动同步给其他访客。纯静态 GitHub Pages 不能让网页直接写回 GitHub，因此不能在公开网页中放仓库 Token。


## 阿里云多人协作方向

当前网站是静态前端，浏览器里的编辑结果只保存在本机，不能自动同步给其他人。将来接入阿里云是可行的，但不建议让前端直接把文件写入服务器；更稳妥的方式是增加一个受保护的后台服务：

- 阿里云 OSS：保存音频文件和封面等大文件。
- 阿里云 RDS 或表格存储：保存歌曲、剧情条目和编辑记录。
- 阿里云 ECS/函数计算 + API 网关：提供登录、权限、上传、保存和发布接口。
- 管理员/编辑者权限：限制谁可以新增、修改、删除和发布内容。
- 操作日志与备份：保留修改人、时间和历史版本，避免多人同时编辑造成覆盖。

建议第一阶段先完成 `assets/audio/` 本地文件发布流程；第二阶段再做阿里云后端。这样不会因为提前接入账号、数据库和上传接口而增加不必要的复杂度。音频文件还需要确认版权或使用授权。


在站内编辑器的“歌单”标签中选择本地音频，可以立即在当前浏览器试听。若要让 GitHub Pages 的所有访客都能听到：将授权音频复制到仓库 `assets/audio/`，点击“生成发布清单 playlist.json”，把清单中的地址改成 `assets/audio/文件名.mp3`，然后把音频和 JSON 一起提交到 GitHub。纯静态 GitHub Pages 不能让网页直接写入仓库，因此不能在公开网页中放 GitHub Token。


在项目目录执行：

```bash
python -m http.server 8080
```

然后访问 `http://localhost:8080/`。
